Network perimeter 2 issues ›

Current gaps

  • Default router admin credentials unchanged on 2 devices
  • UPnP enabled — automatically exposes internal ports

Recommended fixes

  • Enable WPA3 encryption on all Wi-Fi networks
  • Create a separate VLAN or guest network for IoT devices
  • Disable UPnP; use manual port forwarding only
  • Enable DNS-over-HTTPS (DoH) to prevent DNS spoofing
  • Deploy a firewall with deep-packet inspection
Device hardening moderate ›
  • Change default usernames and passwords on every device
  • Disable unused services (Telnet, FTP — prefer SSH/HTTPS)
  • Enable automatic firmware updates or check monthly
  • Use device allowlisting — new devices require approval
  • Physically secure high-value devices (NAS, hub controllers)
  • Disable remote access on devices that don't need it
Data & privacy good ›
  • Encrypt local storage on NAS and hub devices (AES-256)
  • Audit cloud data-sharing settings per device
  • Enable two-factor authentication on all cloud accounts
  • Regularly purge motion/audio logs from cameras and speakers
  • Review app permissions — deny microphone/camera when not essential
Emerging threat mitigations proactive ›
  • Validate firmware hashes before installation
  • Monitor unusual outbound connections from IoT devices
  • Segment AI assistants (Alexa, Google) onto isolated VLAN
  • Use a VPN for remote home access — avoid direct port exposure
  • Subscribe to CVE feeds for your device brands
  • Disable voice-activation when not home using physical mute buttons
Critical

Port scanning / reconnaissance

Detect with network IDS. Alert on >50 SYN packets/sec from a single internal host. Auto-isolate the device and notify the homeowner.

Critical

Botnet enrollment (Mirai)

Watch for outbound IRC or C2 traffic on non-standard ports. Block unknown destinations and reflash device firmware on detection.

High

Credential brute-forcing

Enforce lockout after 5 failed logins. Monitor auth logs. Use fail2ban on hub servers. Push notification to user within 60 seconds.

High

Man-in-the-middle (MITM)

Enable ARP spoofing detection. Enforce HTTPS and TLS 1.3 for all device APIs. Alert on unexpected certificate changes in local traffic.

Medium

Firmware tampering

Cryptographic hash verification before every update. Store expected hashes in a tamper-evident log. Alert if any mismatch is detected.

Medium

Anomalous data exfiltration

Baseline normal traffic volumes per device. Alert if a thermostat uploads >10 MB/day. Use ML-based traffic classification.

Change all default passwords critical

Every router, camera, hub, and smart plug ships with public default credentials. Use a unique 16+ character password per device.

Segment your IoT network critical

Create a separate Wi-Fi VLAN for smart devices. This stops a compromised bulb from accessing your laptop or personal files.

Enable auto-updates on all devices critical

Most attacks exploit known, already-patched vulnerabilities. Automatic updates are the single highest-leverage security action you can take.

Activate two-factor authentication recommended

Enable 2FA on your Google Home, Amazon, Apple Home, and SmartThings accounts. Use an authenticator app rather than SMS.

Disable unused remote access features recommended

If you never control your thermostat remotely, disable that feature. Every open port is an attack surface that doesn't need to exist.

Audit cloud data permissions quarterly routine

Review what each device uploads to vendor clouds. Revoke unnecessary permissions — especially microphone and location access.

Set up a password manager routine

Use Bitwarden or 1Password to generate and store unique credentials for every device and cloud account securely.

Mirai variant targeting TP-Link Archer routers

A new Mirai variant actively scans for TP-Link Archer routers with default credentials on port 23. Update firmware to v1.0.15 or later and change the admin password immediately.

Action: Patch now · Affected: Archer A7, C7, C20

Warning Amazon Ring

Ring cameras — disable public sharing by default

Ring's Neighbours feature shares footage publicly by default. Disable in Privacy Settings → Video Privacy and opt out of Amazon Sidewalk to prevent mesh network enrolment.

Action: Review settings · Affected: All Ring devices
